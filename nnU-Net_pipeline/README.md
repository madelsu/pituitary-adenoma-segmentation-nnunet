# ⚙️ nnU-Net Pipeline

This folder contains example SLURM scripts demonstrating the full **nnU-Net v2 pipeline** — from preprocessing through training to inference — applied to pituitary adenoma segmentation.

The examples use **Dataset 501** (Pandit et al., 2025) as a reference, but the same workflow applies to any dataset by adjusting paths, dataset IDs, and resource allocations.

## 📂 Scripts

| Script | Pipeline Step | Description |
|---|---|---|
| `nnunet_preprocess_501.sh` | Preprocessing | Dataset verification, planning, and preprocessing |
| `train_501_3d_fullres_ds006248.sh` | Training | 3D full resolution training with all folds |
| `predict_501_3d_fullres_ds006248.sh` | Inference | Generate segmentation predictions on the test set |

## 🔧 Environment Setup

All scripts follow the same environment setup pattern:

1. **Load CUDA module** on the HPC cluster
2. **Activate the conda environment** containing nnU-Net v2
3. **Set nnU-Net environment variables** pointing to the project directories

```bash
module load cuda/12.2 2>/dev/null || true
source ~/.bashrc
conda activate nnunet_new

PROJECT_ROOT="/path/to/your/project"

export nnUNet_raw="$PROJECT_ROOT/nnUNet_raw"
export nnUNet_preprocessed="$PROJECT_ROOT/nnUNet_preprocessed"
export nnUNet_results="$PROJECT_ROOT/nnUNet_results"
```

> **Note:** Update `PROJECT_ROOT` to match your own directory structure. The nnU-Net environment variables must be set in every script or session.

## 1️⃣ Preprocessing

**Script:** `nnunet_preprocess_501.sh`

This step verifies dataset integrity, analyzes the data properties, and generates the preprocessing plans that nnU-Net will use for training.

### SLURM Resources

| Resource | Value |
|---|---|
| CPUs | 8 |
| Memory | 80 GB |
| Time | 12 hours |
| GPU | Not required |

### Key Command

```bash
nnUNetv2_plan_and_preprocess -d DATASET_ID -np $SLURM_CPUS_PER_TASK --verify_dataset_integrity
```

The script also includes sanity checks that count the number of training images, labels, and test images before running.

---

## 2️⃣ Training

**Script:** `train_501_3d_fullres_ds006248.sh`

Trains the nnU-Net model using the **3D full resolution** configuration. The `all` fold flag trains on the entire training set (as opposed to individual cross-validation folds).

### SLURM Resources

| Resource | Value |
|---|---|
| CPUs | 4 |
| Memory | 40 GB |
| Time | 48 hours |
| GPU | 1 |

### Key Command

```bash
nnUNetv2_train DATASET_ID 3d_fullres all --npz
```

The `--npz` flag saves softmax predictions alongside the model checkpoints, which are needed for ensemble inference.

### Sanity Checks

Before training starts, the script verifies that:
- Training images and labels exist and are counted
- The preprocessed dataset directory is present

---

## 3️⃣ Inference

**Script:** `predict_501_3d_fullres_ds006248.sh`

Generates segmentation masks for the test set using the trained model.

### SLURM Resources

| Resource | Value |
|---|---|
| CPUs | 8 |
| Memory | 20 GB |
| Time | 6 hours |
| GPU | 1 |

### Key Command

```bash
nnUNetv2_predict \
  -d DATASET_ID \
  -i /path/to/imagesTs \
  -o /path/to/output \
  -c 3d_fullres \
  -f all
```

The `-f all` flag uses all cross-validation folds for ensemble prediction. The script cleans the output folder before running to ensure a fresh prediction.

## 🔁 Adapting to Your Dataset

These scripts are tailored for Dataset 501, but you can easily adapt them to any dataset by modifying:

| Parameter | What to Change |
|---|---|
| `DATASET_ID` | Your nnU-Net dataset ID (e.g., 501, 502, 503) |
| `DATASET_NAME` | Folder name in `nnUNet_raw/` (e.g., `Dataset502_Pituitary`) |
| `PROJECT_ROOT` | Path to your project directory |
| SLURM resources | Adjust memory, time, and CPUs based on dataset size |
| Input/output paths | Update `INP` and `OUT` variables for inference |

> **Tip:** Smaller datasets may need less memory and training time. Larger or higher-resolution datasets (e.g., Černý et al., 2025 with 512×512×336 volumes) may require more resources.

## 📁 Expected Directory Structure

After running the full pipeline, your project directory should look like:

```
project_root/
├── nnUNet_raw/
│   └── DatasetXXX_Pituitary/
│       ├── imagesTr/
│       ├── labelsTr/
│       ├── imagesTs/
│       ├── labelsTs/
│       └── dataset.json
├── nnUNet_preprocessed/
│   └── DatasetXXX_Pituitary/
│       └── ...                    # Auto-generated by preprocessing
├── nnUNet_results/
│   └── DatasetXXX_Pituitary/
│       └── nnUNetTrainer__nnUNetPlans__3d_fullres/
│           └── ...                # Model checkpoints and logs
└── predsTs_XXX_3d_fullres/
    └── *.nii.gz                   # Predicted segmentation masks
```
